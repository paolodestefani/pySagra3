Login procedure

When the application started for the first time, the connection parameters are requested and the window looks like in the following figure:

The parameters required to connect to the database server are:

  • User: type in the application user name
  • Password: type in the user password The user password can be optional based on the authentication settings There are 2 built in user account:
    • system with password System@1
    • utente with password utente
  • Server: enter the name of the computer running the role of database server. It is also possible to enter the server's ip address (eg 192.168.1.1). If the program runs on the same computer where the database server is running You can enter the computer name 'localhost' or the ip 127.0.0.1
  • Port: type in the TCP port to use for the connection. Typically in PostgreSQL installations port 5432 is used, ask the administrator network if a different port is used
  • Database: type in the database name to connect to

The first time you access a server ALL of these parameters must be setted. The subsequent times the same parameters already used successfully are re-proposed excluding the username and password:
