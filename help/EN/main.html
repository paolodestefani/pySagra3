Overview

pySagra stems from the need to manage the gastronomic stand of my village's festival. The activity to be managed is therefore the order entry of products, typically first courses, main courses, drinks, etc., the subsequent distribution of the orders, real work orders, to the various departments of the stand, for example bar, restaurant, but also pizzeria, ice cream parlor, etc. The orders are placed in the cash desks and must be the most quick and simple as possible. Once the order has been entered it is divided by department and distributed to the departments concerned. Once the department copy is printed the order it is considered automatically processed. There is no management of order fulfillment as it is superfluous for the type of activity that is carried out. Apart from this the program allows the closure of the cash register end various sales statistics. All lists, statistics including, they are printable and automatically exportable to Excel. Currently, fiscal printing is not managed.

The features of the application can be summarized in the following points:

  • Management of events : all operations come always referring to a specific event (eg. Sagra del Pepper, Beer Festival, etc.).
  • Management of departments (eg Bar, restaurant, Pizzeria, etc.). Some departments may not be available for specific events.
  • Managing sales items, distinguishing them in elementary items, kits (eg three of a kind: sausages + ribs + chop) and menu (ex. sagra menu: beer + tris + fried potatoes). The kits must be composed by themselves elementary articles. Menus may contain kits.
  • Management of variants of items (eg pizza margherita without mozzarella, with salamino, etc.). The variant may provide a price increase.
  • Delivery management at table or per Take . It is possible to check the existence of the table number entered.
  • stock level control: for elementary articles which interests to monitor it is possible to insert the stock available. The program automatically updates the availability based on sales made.
  • Management of sales orders through a optimized form aimed at achieving the maximum insertion speed with stock control of the articles, total calculation, amount cashed, change, etc.
  • Generation of the order printing for the customer, for the various departments concerned and one for covers. All these prints are in A5 format and optional
  • Print of the cashed per day and half a day
  • Management of various sales statistics even comparing different events
  • All archives (events, departments, items, etc.) and the available statistics can be viewed, printable A4 and exportable in excel
  • The application is cross-platform : developed and tested on Windows and Linux, it should work correctly also on Mac OSX

Application architecture

This program was developed with in mind some specific objectives:

  • Deepen my knowledge of the Python language
  • Implement an application with a graphical interface e based on a sql database server
  • Implement a multiuser application with client / server architecture
  • The application must be multi-platform and available low / no cost to anyone who requests it

Consequently for the realization of this application have been used:

  • The Python programming language version 3.7
  • The Qt libraries version 5.x
  • The Qt wrapper libraries PyQt version 5.x
  • The PostgreSQL database server version 10.x
  • The psycopg Python connector to PostgreSQL version 2.8.x
  • The oxygen icons icon set
  • The pySagra icon is from DelliOS System Icons set of Wendell Fernandes

All the tools listed are multiplatform and available for free
